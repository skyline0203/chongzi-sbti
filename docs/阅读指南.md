# 艾斯BTI - 阅读指南

> 写给翀子的代码阅读路线图
>
> "从零开始，但不止于零"

---

## 🎯 总原则

你不需要一行一行读懂所有代码。**先理解整体流程，再深入细节。**

就像修车——先知道发动机、变速箱、轮子各是干嘛的，再去看螺丝怎么拧。

---

## 📖 推荐阅读顺序

### 第一步：先看文档（30分钟）

1. **需求文档** — 知道这个项目要干什么
2. **本文件** — 知道怎么读代码
3. **代码目录结构** — 知道每个文件是什么

### 第二步：理解界面（20分钟）

先不看代码，打开微信开发者工具，**用眼睛看**：

1. 打开小程序，看首页长什么样
2. 点「开始鉴定」，看答题页长什么样
3. 做完题，看结果页长什么样

**重点观察**：每个页面上有什么元素（标题、按钮、选项等）

### 第三步：从简单到复杂读代码（1小时）

#### 🟢 入门级：从结果文案开始
```
utils/results.js
```
- 这是纯数据文件，没有任何逻辑
- 看看 4 种类型的文案是怎么写的
- 你会发现就是一个个"字典"，key 对应 value

#### 🟢 入门级：看题库
```
utils/questions.js
```
- 也是纯数据
- 15 个对象，每个有 `text`（题目）和 `options`（选项）
- 每个选项有个 `type`（S/H/I/T）

#### 🟡 初级：看首页
```
pages/index/index.wxml  ← 先看界面
pages/index/index.js    ← 再看逻辑
pages/index/index.wxss  ← 最后看美化
```
- 首页逻辑最简单：就一个按钮，点击跳转

#### 🟡 初级：看答题页
```
pages/quiz/quiz.wxml
pages/quiz/quiz.js
```
- 这是最核心的逻辑页
- 重点关注 `selectOption` 和 `nextQuestion` 函数

#### 🟠 中级：看结果页
```
pages/result/result.wxml
pages/result/result.js
```
- 涉及广告逻辑和解锁机制
- 比较复杂，可以先跳过广告部分

#### 🔴 高级：看登录和管理（新增功能）
```
pages/login/login.js
```
- 涉及微信授权、网络请求
- 等前面都理解了再看

## 🧠 理解代码的心法

### 1. 不要死磕每一行

```javascript
// 看到这种代码，知道它是干嘛的就行，不需要理解每一步
wx.createRewardedVideoAd({ adUnitId: adUnitId });
```

**翻译成人话**：这是创建一个广告对象。

### 2. 看注释，不看代码

我在代码里加了大量中文注释。**先读注释**，注释读不懂再看代码。

### 3. 遇到不懂的函数，查微信官方文档

- 微信小程序官方文档：https://developers.weixin.qq.com/miniprogram/dev/framework/
- 搜索函数名就行，比如搜 `wx.navigateTo`

### 4. 用"翻译"思维

把代码翻译成人话：

```javascript
// 代码
const answers = [...this.data.answers];
answers[this.data.currentIndex] = type;

// 翻译：把之前选的答案复制一份，然后把当前这题的答案改成刚刚选的那个
```

## ❓ 常见疑问速查

| 你可能会问 | 答案 |
|-----------|------|
| `this` 是什么？ | 指代当前页面本身，就像"我自己" |
| `setData` 是干嘛的？ | 修改页面数据，页面会自动刷新显示 |
| `{{变量}}` 是什么？ | 把 JS 里的数据"搬到"页面上显示 |
| `rpx` 和 `px` 什么区别？ | rpx 会自动适配不同手机屏幕，推荐用 rpx |
| `bindtap` 是什么？ | 点击事件，相当于"当用户点击时做某事" |
| `e.currentTarget.dataset` 是什么？ | 从被点击的元素上拿到自定义数据 |
| 为什么要 `...` 三点？ | 展开运算符，复制数组用的，防止修改原数据 |
| `module.exports` 是什么？ | 把这个文件的东西"导出"给别人用 |

## 🔧 如何动手改代码

### 最安全的实验区：改文案

打开 `utils/results.js`，随便改几个字，保存，看页面变化。

### 进阶实验：改题目

打开 `utils/questions.js`，加一道题，保存，看效果。

### 高级实验：改样式

打开任意 `.wxss` 文件，改颜色、大小，看页面变化。

**改坏了？不要慌！** 用 `git checkout` 恢复，或者直接跟我说。
