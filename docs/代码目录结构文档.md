## 🔑 关键文件解释

### app.js — 小程序的"总开关"
> 类比：你家的电闸总开关

- 小程序启动时第一个执行的文件
- 可以在这里定义全局变量（比如广告ID）
- `onLaunch()` 函数 = 小程序启动时自动运行的代码

### app.json — 小程序的"配置表"
> 类比：一本书的目录

- `pages` 数组 = 告诉微信有哪些页面
- `window` 对象 = 设置导航栏颜色、标题等
- 第一个 page 就是打开小程序时显示的页面

### .js 文件 — 逻辑代码
> 类比：大脑，负责思考和决策

- 处理用户操作（点击、滑动等）
- 控制数据变化
- 页面跳转

### .wxml 文件 — 界面结构
> 类比：骨架，决定页面上有什么

- `<view>` 类似 HTML 的 `<div>`（一个方块容器）
- `{{变量名}}` = 把 JS 中的数据显示到页面上
- `bindtap="函数名"` = 点击时触发哪个函数

### .wxss 文件 — 样式美化
> 类比：衣服，决定长什么样

- 跟 CSS 一模一样，就是换个后缀名
- `rpx` 是微信特有的单位，自动适配不同手机

### .json 文件 — 配置
> 类比：个人设置

- 每个页面可以有自己的配置
- 比如设置导航栏标题

## 📐 文件之间的关系

```
用户点击按钮
    ↓
.wxml 中的 bindtap 触发
    ↓
.js 中的函数执行
    ↓
函数修改 data 中的数据
    ↓
.wxml 中的 {{变量}} 自动更新显示
```

**简单说：WXML 是界面，JS 是逻辑，WXSS 是美化，JSON 是配置。**
